### 背景

`--amend`や`squash`でvimを起動してコメント入力する場合、

チケット番号などで用いる`#(シャープ)`を先頭に入力すると、コメント行扱いとなってしまいます。。

### 解決！

```
git commit --amend -m "#チケット番号"
```

上記コマンドで直前のコミットメッセージを変更すると、先頭の`#(シャープ)`が通ります。


直前以外のメッセージを変更する際は・・・。

`git rebase -i`でコミット順を入れ替えて、メッセージ変えて、また順番戻すで実現できそうだがめんどくさい。
